# Test Plan: BRD-APP-PCA-01

## Overview
This test plan covers the functionality of updating quantity and removing items in the cart on the e-commerce application, ensuring that users can modify their cart contents as needed.

## Scope
In scope:
- Updating quantity of items in the cart
- Removing items from the cart
Out of scope:
- Adding items to the cart
- Checkout process
- Payment gateway integration

## Test Scenarios

### Scenario 1: Update Quantity of Item in Cart
**Type:** Happy Path
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in their cart
- The item quantity is greater than 1
**Steps:**
1. Given the user is on the cart page, they click on the "Edit" button next to an item.
2. When the user enters a new quantity (e.g., 2) and clicks "Update".
**Expected Result:** The item quantity is updated to the new value, and the subtotal and total are recalculated accordingly.

### Scenario 2: Remove Item from Cart
**Type:** Happy Path
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in their cart
**Steps:**
1. Given the user is on the cart page, they click on the "Remove" button next to an item.
2. When the user confirms the removal.
**Expected Result:** The item is removed from the cart, and the subtotal and total are updated.

### Scenario 3: Update Quantity to Zero
**Type:** Edge Case
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in their cart
**Steps:**
1. Given the user is on the cart page, they click on the "Edit" button next to an item.
2. When the user enters a quantity of 0 and clicks "Update".
**Expected Result:** The item is removed from the cart, and the subtotal and total are updated.

### Scenario 4: Remove Last Item from Cart
**Type:** Edge Case
**Preconditions:**
- The user is logged in to the application
- The user has only one item in their cart
**Steps:**
1. Given the user is on the cart page, they click on the "Remove" button next to the item.
2. When the user confirms the removal.
**Expected Result:** The item is removed from the cart, the cart is empty, and the user is notified that their cart is empty.

### Scenario 5: Update Quantity of Non-Existent Item
**Type:** Negative
**Preconditions:**
- The user is logged in to the application
- The user has no items in their cart
**Steps:**
1. Given the user is on the cart page, they attempt to update the quantity of an item.
**Expected Result:** An error message is displayed, indicating that there are no items in the cart to update.

### Scenario 6: Remove Non-Existent Item
**Type:** Negative
**Preconditions:**
- The user is logged in to the application
- The user has no items in their cart
**Steps:**
1. Given the user is on the cart page, they attempt to remove an item.
**Expected Result:** An error message is displayed, indicating that there are no items in the cart to remove.
