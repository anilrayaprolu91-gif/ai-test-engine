# Test Plan: BRD-APP-PCA-01

## Overview
This test plan covers the functionality of updating quantity and removing items in the cart on the application website https://practicesoftwaretesting.com.

## Scope
In scope:
- Updating quantity of items in the cart
- Removing items from the cart
Out of scope:
- Adding items to the cart
- Checkout process
- Payment gateway

## Test Scenarios

### Scenario 1: Update Quantity of Item in Cart
**Type:** Happy Path
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in the cart
**Steps:**
1. Given the user is on the cart page, when they click on the quantity dropdown of an item, then a list of available quantities should be displayed.
2. When the user selects a new quantity from the dropdown, then the update cart button should be enabled.
3. When the user clicks the update cart button, then the quantity of the item should be updated successfully.
**Expected Result:** The quantity of the item in the cart is updated, and the total cost is recalculated accordingly.

### Scenario 2: Remove Item from Cart
**Type:** Happy Path
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in the cart
**Steps:**
1. Given the user is on the cart page, when they click on the remove button next to an item, then a confirmation prompt should be displayed.
2. When the user confirms the removal, then the item should be removed from the cart.
**Expected Result:** The item is removed from the cart, and the total cost is updated accordingly.

### Scenario 3: Update Quantity to Zero
**Type:** Edge Case
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in the cart
**Steps:**
1. Given the user is on the cart page, when they update the quantity of an item to zero, then the item should be removed from the cart.
**Expected Result:** The item is removed from the cart, and the total cost is updated accordingly.

### Scenario 4: Remove Last Item from Cart
**Type:** Edge Case
**Preconditions:**
- The user is logged in to the application
- The user has only one item in the cart
**Steps:**
1. Given the user is on the cart page, when they click on the remove button next to the last item, then the cart should be empty.
2. When the cart is empty, then a message indicating that the cart is empty should be displayed.
**Expected Result:** The cart is empty, and a message indicating that the cart is empty is displayed.

### Scenario 5: Update Quantity of Non-Existent Item
**Type:** Negative
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in the cart
**Steps:**
1. Given the user is on the cart page, when they try to update the quantity of an item that does not exist in the cart, then an error message should be displayed.
**Expected Result:** An error message is displayed, indicating that the item does not exist in the cart.

### Scenario 6: Remove Non-Existent Item from Cart
**Type:** Negative
**Preconditions:**
- The user is logged in to the application
- The user has at least one item in the cart
**Steps:**
1. Given the user is on the cart page, when they try to remove an item that does not exist in the cart, then an error message should be displayed.
**Expected Result:** An error message is displayed, indicating that the item does not exist in the cart.
