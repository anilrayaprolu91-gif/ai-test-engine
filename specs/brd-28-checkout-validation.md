# Test Goal

Validate required-field and invalid-input error handling during checkout.

# Requirement

User sees validation errors for missing or invalid checkout inputs.

# Acceptance Criteria

1. Submitting empty required fields shows validation messages.
2. Invalid format inputs show field-level errors.
3. Errors are visible and understandable.
4. Correcting inputs clears relevant validation errors.
